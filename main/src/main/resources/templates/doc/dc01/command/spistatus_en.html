Description

The circuit board includes SPI modules such as LORA and TM7705. Additionally, other SPI modules can be externally connected to the board. This command allows you to check the operating status of SPI-related devices.

Parameter Description
Parameter Description Required Type Bytes Input/Output
operate Fixed value: 4 Yes Number 1 Input
Example
Get SPI operating status:

operate=4

Response

spi_bus 1 indicates successful initialization, 0 indicates failure
0.noname:inited 0 represents SPI device 0. A total of 5 SPI devices are supported, numbered 0-4. Device 0 is for the LORA module on the board, device 1 is for the TM7705 sensor, device 2 is for the W5500 chip, and devices 3 and 4 can be used for external modules.
0.noname:inited The "name" indicates whether a name was assigned during the initialization of each module. If no name was assigned, it defaults to "noname."
0.noname:inited=0 Indicates that SPI device number 0, which corresponds to the LORA module, has not been initialized or enabled.
0.noname:inited=0,tx=0,rx=0,fail=0; "tx" and "rx" represent the number of transmissions and receptions, respectively, and "fail" represents the number of failures. Since the device was not initialized, all values are 0.
1.not set:inited=1,tx=12,rx=17,fail=0; SPI device number 1, corresponding to the TM7705, has been initialized and has transmitted 12 times, received 17 times, with 0 failures.
